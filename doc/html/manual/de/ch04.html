Das Event-Modell

Der Application-Server arbeitet ereignisorientiert. Er vermittelt Nachrichten, sog. Ereignisse zwischen dem Webserver und der Anwendung sowie Teilen der Anwendung untereinander.

Ereignisse bestehen aus Objekten der Klasse event und enthalten den Namen des Empfängers sowie beliebig viele Argumente, die durch ihre Namen identifiziert werden.

Events können serialisiert und zu einem beliebigen späteren Zeitpunkt ausgelöst werden. Der Empfänger besteht aus einer gewöhnlichen benutzerdefinierten PHP-Funktion, der als einziger Parameter eine Referenz auf die Applikationsklasse übergeben wird. Über sie können die Event-Argumente abgerufen und neue Events erzeugt werden.

Das erste Event wird beim Aufruf der Anwendung aus der URL generiert, oder mit Hilfe eines darin enthaltenen, einmaligen Schlüsselns, einem Token, aus der Datenbank abgerufen und ausgelöst. Danach können neue URLs aus Events erzeugt werden.
