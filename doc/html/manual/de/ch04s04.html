Subsessions

application bietet ausser einer globalen Session, in der Daten eventübergreifend abgelegt werden können auch sog. Subsessions, die mit dem Event-Mechanismus gesteuert werden. Beim ersten Aufruf einer Anwendung wird automatisch eine leere Subsession geöffnet.

Ein Event öffnet vor dem versenden der Argumente eine neue Subsession, wenn dem Event ein Absender mitgeteil wurde, zu dem zurückgekehrt werden kann.

Um Daten in der Subsession abzulegen muss die Methode application::arg() verwendet werden.

Das Flag ARG_SUB sorgt dafür, daß ein erfolgreich gelesenes Argument in der aktuellen Subsession abgelegt wird. Wurde das Argument nicht per Event übergeben, so wird der zuletzt in der Subsession abgelegte Wert verwendet. Liegt auch dort das Argument nicht vor, wird der Programmablauf mit einer Fehlermeldung beendet, es sei denn, auch das Flag ARG_OPTIONAL wurde gesetzt.

Der Event-Handler 'return2caller' verwirft die aktuelle Subsession und stellt die Argumente der vorherigen Subsession wieder her.
