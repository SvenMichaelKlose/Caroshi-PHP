Links und Formularelemente im Cursor-Context

admin_panel kann mit open_context() die Referenz auf einen Cursor übergeben werden der für jedes Event in URLs und Form-Namen neu serialisiert wird. Form-Widgets wird der Name eines Feldes im Record auf den der Cursor zeigt übergeben (abrufbar mit cursor::field()).

Der Context-Cursor wird von url() und new_formfield() im Event-Argument '_cursor' sowie im der Klassenvariable 'cursor' von _form_element-Objekten abgelegt.

Kontexte können verschachtelt werden.
